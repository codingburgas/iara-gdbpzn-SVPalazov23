## 🎯 Мисия и Цел на Проекта
Основната цел на приложението е да дигитализира и автоматизира ежедневните процеси в **Изпълнителната агенция по рибарство и аквакултури (ИАРА)**. Системата предоставя централизиран уеб интерфейс за инспектори и администратори, с който лесно и безгрешно се управлява риболовният флот, издават се разрешителни по регулации на ЕС и се изчисляват такси за любителски риболов.

## 🛠️ Какво съм използвал?

За разработването на тази система заложих на съвременни, бързи и надеждни технологии:

| Компонент | Технология | Описание и роля в проекта |
| :--- | :--- | :--- |
| **Backend** | 🐍 **Python 3.x** | Основният език, на който е написана цялата бизнес логика. |
| **Уеб Рамка** | 🌶️ **Flask** | Лек и мощен софтуерен рамков инструмент за маршрутизация (routing) и обработка на потребителски заявки. |
| **База Данни** | 🗄️ **SQLite 3** | Релационна база данни. Всички данни се пазят сигурно в един локален файл (`iara.db`), като се използват релационни връзки (`FOREIGN KEY`) за интегритет. |
| **Пакетен Мениджър**| ⚡ **`uv` (by Astral)** | Използван за светкавично управление на зависимостите и изолиране на проекта във виртуална среда (`.venv`). |
| **Шаблонизатор** | 🎨 **Jinja2** | Позволява динамично генериране на HTML страници и директно вграждане на Python данни в изгледите. |
| **Frontend** | 🌐 **HTML5 & CSS3** | Чист, семантичен код за интерфейса с интуитивна синя навигационна лента, съобразена с брандинга на държавните институции. |
| **Контрол на версиите**| 🐙 **Git & GitHub** | Използвани за системно проследяване на промените и сигурно качване на кода в GitHub Classroom. |

## 📦 Архитектура и Структура на Файловете

Проектът следва чиста модулна структура, вдъхновена от архитектурния модел MVC (Model-View-Controller):

```text
├── database/
│   └── iara.db           # Локален SQLite файл (автоматично генериран)
├── static/
│   ├── css/
│   │   └── style.css     # Глобални стилове на приложението
│   └── images/           # Графични активи и лога
├── templates/
│   ├── base.html         # Главен родителски шаблон (Header, Меню, Footer)
│   ├── home.html         # Начално информационно табло
│   ├── ships.html        # Интерфейс на Регистъра за кораби
│   ├── permits.html      # Интерфейс за Издаване на разрешителни
│   └── tickets.html      # Интерфейс за Любителски билети
├── app.py                # Сървърно ядро и Контролер (Маршрути и логика)
├── models.py             # Дефиниции на схемата на базата данни (Таблици)
└── README.md             # Документация на проекта (Този файл)
